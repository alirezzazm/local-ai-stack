# local-ai-stack

راه‌اندازی یک هوش مصنوعی محلی (LLM) با Ollama در **سه حالت سخت‌افزاری**.
هرجا این ریپو را clone کنی، یک اسکریپت همه‌چیز را خودکار نصب می‌کند و مدل/دیتاست را دانلود می‌کند.

> A self-hostable local LLM stack with three hardware profiles. Clone anywhere → one script installs Ollama, pulls the right model, and downloads the dataset.

---

## سه حالت اجرا (Profiles)

| حالت | سخت‌افزار هدف | مدل پیش‌فرض | RAM/VRAM لازم |
|------|----------------|-------------|----------------|
| `weak`   | لپ‌تاپ / VPS بدون GPU (CPU only) | `llama3.2:3b`     | ~۸ گیگ RAM |
| `strong` | سیستم قوی با یک GPU (مثل RTX 4090 / 24GB) | `qwen3.6:27b` (Q4) | ~۲۴ گیگ VRAM |
| `server` | سرور خیلی قوی / چند GPU | `qwen3.6:27b` (full) یا `llama3.3:70b` | ۴۸ گیگ+ VRAM |

پروفایل‌ها در پوشه‌ی [`profiles/`](profiles/) به‌صورت JSON تعریف شده‌اند و به‌راحتی قابل ویرایش‌اند.

---

## نصب و اجرا

### ویندوز (PowerShell)
```powershell
# حالت ضعیف (پیش‌فرض)
./scripts/setup.ps1 -Profile weak
./scripts/run.ps1   -Profile weak

# حالت قوی
./scripts/setup.ps1 -Profile strong
```

### لینوکس / مک (bash)
```bash
chmod +x scripts/*.sh
./scripts/setup.sh weak     # یا strong / server
./scripts/run.sh weak
```

اسکریپت `setup` این کارها را انجام می‌دهد:
1. اگر Ollama نصب نیست، نصبش می‌کند.
2. سرویس Ollama را بالا می‌آورد.
3. مدلِ متناظر با پروفایل را `pull` می‌کند.
4. دیتاست را در `data/` آماده می‌کند.

## اقدامات (DAZ کار انجام می‌دهد — مثل جارویس)
با تیکِ **⚙️ اقدامات** در UI، DAZ می‌تواند ابزار اجرا کند (Ollama tool-calling). ابزارها در [`webui/tools.js`](webui/tools.js):

**اطلاعات:** ساعت/تاریخ · آب‌وهوا · جستجوی وب · اخبار · ویکی‌پدیا · قیمت رمزارز/ارز · محاسبه‌گر · تبدیل واحد
**بهره‌وری:** یادآور/تایمر · یادداشت/لیست کار · تقویم
**سیستم:** اطلاعات سیستم · جستجوی فایل · خواندن/نوشتن فایل · صدا · روشنایی · باز کردن برنامه/سایت
**رسانه:** کنترل پخش · یوتیوب
**محافظت‌شده (نیاز به `DAZ_ALLOW_SYSTEM=1`):** قفل/خاموش/ری‌استارت · اجرای دستور PowerShell
**نیاز به تنظیم:** ایمیل (`DAZ_SMTP`) · تلگرام (`DAZ_TG_TOKEN`) · خانه‌ی هوشمند

پیاده‌سازی در `/api/agent`: مدل تصمیم می‌گیرد کدام ابزار را صدا بزند، سرور اجرا می‌کند و نتیجه را به مدل برمی‌گرداند.

ابزارها **گروه‌بندی** شده‌اند (۸ ابزارِ کلی به‌جای ۲۷ تا) تا انتخابِ آن‌ها روی مدل کوچک محلی قابل‌اطمینان باشد: `get_info`، `calculate`، `convert_units`، `manage_data`، `control_system`، `open_thing`، `file_op`، `run_command`.

> اگر نمی‌خواهی DAZ کاری اجرا کند، تیک «اقدامات» را بردار. برای فعال‌کردن اقدامات خطرناک: `DAZ_ALLOW_SYSTEM=1`.

## مأموریت — تصمیم‌گیری خودمختار (Agent)
دکمه‌ی **🧠 مأموریت**: یک **هدف** می‌دهی و DAZ خودش گام‌به‌گام تصمیم می‌گیرد، چند ابزار را پشت‌سرهم اجرا می‌کند و نتیجه را می‌دهد (حلقه‌ی decide → act → observe در `/api/think`). مثال: «هوای تهران را بگو و اگر آفتابی بود یک یادآور بگذار». مراحل تصمیم‌گیری در پنل نمایش داده می‌شوند.
> همین حلقه نقشِ **مشاور** را هم دارد (با دانشِ آموخته گزینه‌ها را می‌سنجد و توصیه می‌دهد). تصمیم‌گیریِ خودمختار روی مدل کوچک محلی محدود است؛ روی GPU قوی‌تر می‌شود.

## شخصی‌سازی (جارویسِ تو)
دکمه‌ی **👤 شخصیت** در UI یک پنل باز می‌کند تا DAZ را شخصیِ خودت کنی:
- **نام تو** و اینکه **چطور صدایت بزند** (مثلاً «قربان»، اسم کوچک، یا «رفیق»).
- **لحن و شخصیت**: جارویسی کلاسیک / جدی و حرفه‌ای / گرم و صمیمی.
- **نکاتی که همیشه درباره‌ی تو بداند** (شغل، ترجیحات…).

این تنظیمات در `persona.json` ذخیره (و خودکار به git فرستاده) می‌شوند و در **هر گفتگو** به‌عنوان هویت DAZ تزریق می‌شوند — بدون نیاز به ساخت دوباره‌ی مدل. نتیجه: یک دستیار شخصیِ وفادار، مثل جارویس برای تونی استارک.

## دستیار فارسی (DAZ)
یک مدل سفارشی فارسی‌زبان بر پایه‌ی Qwen، با شخصیت و قانون «همیشه فارسی جواب بده» (تعریف در [`modelfiles/daz.Modelfile`](modelfiles/daz.Modelfile)).

```powershell
ollama pull qwen2.5:3b      # مدل پایه
./scripts/build-fa.ps1      # ساخت DAZ   (لینوکس/مک: ./scripts/build-fa.sh)
ollama run daz
```
> نکته: مدل پایه‌ی ۳B فارسی را در حد قابل‌قبول می‌فهمد ولی گرامرش گاهی می‌لنگد. برای فارسیِ روان، روی سیستم GPU‌دار پایه را به `qwen2.5:7b` یا بزرگ‌تر تغییر بده (خط `FROM` در Modelfile).

## رابط گرافیکی (Web UI)
یک رابط چت فارسی (RTL) داخل مرورگر — با ورودی **متن** و **صدا** (تبدیل گفتار به متن توسط مرورگر) و خروجی صوتی (TTS). قلاب ورودی **عکس** هم هست که با یک مدل بینایی (مثل `llava`) فعال می‌شود.

```powershell
# نیاز: Node.js + یک Ollama در حال اجرا + مدل daz ساخته‌شده
./scripts/webui.ps1            # لینوکس/مک: ./scripts/webui.sh
# سپس در مرورگر باز کن:  http://localhost:8080
```
- بدون هیچ وابستگی npm (فقط Node خام). فایل‌ها در [`webui/`](webui/).
- نام مدل را از بالای صفحه می‌توانی عوض کنی (پیش‌فرض `daz`).
- 🎤 ورودی صوتی فارسی در مرورگرهای مبتنی بر Chrome بهتر کار می‌کند.
- 📷 برای فهم عکس به مدل بینایی نیاز است: `ollama pull llava` و سپس نام مدل را در UI به `llava` تغییر بده.

## جستجو و یادگیری (RAG زنده)
دکمه‌ی **🌐 یادگیری** در بالای UI یک پنل با سه راه باز می‌کند:
1. **جستجوی موضوع** در اینترنت (DuckDuckGo، بدون کلید API) — تا ۵ صفحه می‌خواند.
2. **لینک مستقیم** — یک URL می‌دهی و فقط همان صفحه را یاد می‌گیرد.
3. **متن یا فایل** — متن می‌چسبانی یا فایل متنی (`.txt/.md/.csv/.json`) بارگذاری می‌کنی.

در هر سه حالت DAZ خلاصه‌ی فارسی می‌سازد و در `knowledge/` ذخیره می‌کند. از آن به بعد وقتی سؤال مرتبط بپرسی، آن دانش خودکار به مدل داده می‌شود (RAG).

**🔄 آپدیت اطلاعات:** دکمه‌ای در پنل یادگیری که هر موضوعِ آموخته‌شده از اینترنت را **دوباره جستجو و به‌روز** می‌کند تا دانش کهنه نشود.

> سرعت: خلاصه‌سازی/یادگیری از مدل سبک‌تر (`qwen2.5:3b`، قابل تغییر با `DAZ_SUMMARY_MODEL`) استفاده می‌کند تا روی CPU سریع‌تر باشد؛ چت همچنان با مدل بزرگ‌تر کیفیت بالا دارد. روی CPU هر موضوع چند دقیقه طول می‌کشد؛ روی GPU بسیار سریع‌تر.
- آموخته‌ها به‌صورت فایل `.md` در پوشه‌ی [`knowledge/`](knowledge/) می‌مانند (داده‌ی شخصی — در git نادیده گرفته می‌شود).
- یادگیری روی CPU کند است (مدل باید صفحات را خلاصه کند) ولی بدون GPU کار می‌کند.

## بازبینی و تأیید پاسخ‌ها (Human-in-the-loop)
دکمه‌ی **✅ بازبینی** در بالای UI: یک موضوعِ آموخته را انتخاب می‌کنی → DAZ ۵ **پرسش و پاسخ** می‌سازد → هر پاسخ را می‌خوانی، در صورت نیاز **اصلاح** می‌کنی و **✅ درست / ❌ غلط** می‌زنی. با ذخیره:
- پاسخ‌های تأییدشده به‌صورت دانشِ «تأییدشده» در `knowledge/` می‌مانند (RAG از همین‌ها استفاده می‌کند).
- و **به `data/dataset.jsonl` افزوده می‌شوند** — یعنی به‌مرور یک دیتاستِ آموزشیِ تأییدشده‌ی انسانی می‌سازی که بعداً برای fine-tune استفاده می‌شود.

## همگام‌سازی خودکار با GitHub
هر چیزی که یاد بگیرد یا تأیید کنی، سرور **خودکار** آن را commit و push می‌کند:
- بعد از هر `learn` یا `verify`، فایل‌های `knowledge/` و `data/dataset.jsonl` به GitHub فرستاده می‌شوند.
- از اعتبارنامه‌ی گیتِ همان سیستم استفاده می‌کند (روی سیستم خودت: یک‌بار `gh auth login` یا Git Credential Manager کافی است).
- اگر گیت احراز هویت نشده باشد، بی‌سر‌و‌صدا رد می‌شود (خطا نمی‌دهد).
- برای خاموش‌کردن: متغیر محیطی `DAZ_GIT_SYNC=0` را قبل از اجرای سرور تنظیم کن.

> این یعنی هرجا ریپو را clone کنی، آموخته‌های قبلی هم همراهش هستند.

## دسترسی از هر جا (مثل Claude) + اپ نصب‌شدنی
DAZ را روی یک سرور اجرا کن و از مرورگر یا اپِ نصب‌شده‌ی هر دستگاهی به آن وصل شو.

### ۱) سرور را در دسترس شبکه بگذار + رمز بده
```powershell
$env:HOST="0.0.0.0"          # روی همه‌ی اینترفیس‌ها گوش بده (پیش‌فرض همین است)
$env:DAZ_TOKEN="یک‌رمز‌قوی"   # ⚠️ حتماً قبل از افشا تنظیم کن
./scripts/webui.ps1
```
هر دستگاه دیگری در همان شبکه: `http://IP-سرور:8080` را باز کند و یک‌بار توکن را وارد کند (در مرورگر ذخیره می‌شود).

### ۲) دسترسی از اینترنت (بدون IP ثابت)
ساده‌ترین راهِ امن، یک تونل است (HTTPS رایگان و بدون باز کردن پورت):
```bash
cloudflared tunnel --url http://localhost:8080
```
آدرس `https://...trycloudflare.com` که می‌دهد را هر جا باز کن. (HTTPS برای نصبِ اپ روی موبایل لازم است.)

### ۳) نصب به‌عنوان اپ (PWA)
در مرورگر (Chrome/Edge موبایل یا دسکتاپ) → منو → **Install / Add to Home Screen**. DAZ مثل یک اپ مستقل باز می‌شود.

### امنیت ⚠️
- بدون `DAZ_TOKEN` هرکسی که به پورت برسد می‌تواند از DAZ استفاده کند (و اگر `DAZ_ALLOW_SYSTEM=1` باشد، سیستم را کنترل کند). **همیشه هنگام افشا توکن بگذار.**
- روی ویندوز باید پورت ۸۰۸۰ را در فایروال باز کنی (یا از تونل استفاده کنی).
